# 🔐 Zero Trust কী?

👉 Simple rule:

```text
❌ Trust nothing
✔ Verify everything
```

👉 Meaning:

* User trusted না
* Service trusted না
* Network trusted না

👉 Every request → verify করতে হবে

# 🔥 4 Layer Security (Must follow)

---

## 🔹 1. 🔐 Identity (Keycloak)

👉 Every request must have identity

### Flow:

```text
User → Keycloak → JWT token
```

👉 তারপর:

```text
Client → Gateway → (JWT)
```

### Rule:

* No token = ❌ reject
* Invalid token = ❌ reject

---

## 🔹 2. 🔒 Authentication (gRPC Metadata + JWT)

👉 gRPC-তে token যায় **Metadata header** দিয়ে

### Example (Client side):

```java
Metadata metadata = new Metadata();
metadata.put(
    Metadata.Key.of("Authorization", Metadata.ASCII_STRING_MARSHALLER),
    "Bearer " + token
);
```

👉 Interceptor দিয়ে attach করো

---

### Server side (validate token):

* JWT decode
* Keycloak public key দিয়ে verify

---

## 🔹 3. 🔑 Authorization (Role-based access)

👉 শুধু login enough না

### Example:

```text
ROLE_ADMIN → debit allowed
ROLE_USER → only read balance
```

👉 gRPC method level check:

```java
if (!hasRole("ADMIN")) {
    throw Status.PERMISSION_DENIED.asRuntimeException();
}
```

---

## 🔹 4. 🔐 mTLS (Service-to-Service Security)

👉 Service ↔ Service communication secure করতে হবে

### Without mTLS:

```text
user-service → account-service ❌ (anyone call করতে পারে)
```

### With mTLS:

```text
user-service ⇄ account-service ✔ (certificate verify)
```

👉 Both side verify:

* Client certificate
* Server certificate

---

# ⚡ Full Zero Trust Flow (Your Project)

```text
User → Keycloak (login)
      ↓
JWT token
      ↓
Gateway (verify token)
      ↓
user-service
      ↓ (JWT + mTLS)
account-service (verify again)
      ↓
response
```

# 🚨 Common Mistakes (Avoid)

❌ Only gateway validate → backend verify না
❌ No mTLS
❌ Hardcoded trust between services
❌ No role check
❌ No logging

```text
Every request must be:

✔ Logged (Splunk)
```
