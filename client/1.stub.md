# 🧠 What is a Stub?

👉 **Stub = Auto-generated client proxy**

👉 Simple definition:

> A **stub is a local Java object that lets you call a remote gRPC service as if it were a normal method call.**

---

# 🎯 Real-Life Analogy

Think like this:

* 📞 You call a friend → you don’t go physically
* ☎️ Phone = **Stub**

👉 You call:

```java
accountStub.getAccount(request);
```

But actually:

* It goes over network 🌐
* Talks to another service
* Returns response

---

# ⚡ In Your Code

```java
@GrpcClient("account-service")
private AccountServiceGrpc.AccountServiceBlockingStub accountStub;
```

👉 This `accountStub` is:

✔ Not real service
✔ Not your class
✔ It is **generated automatically from `.proto`**

---

# 🔥 How Stub is Created

## Step 1: You write `.proto`

```proto
service AccountService {
  rpc GetAccount (AccountRequest) returns (AccountResponse);
}
```

---

## Step 2: gRPC generates Java code

```java
AccountServiceGrpc.java
```

Inside it:

```java
public static final class AccountServiceBlockingStub {
    public AccountResponse getAccount(AccountRequest request) {
        // internally makes network call
    }
}
```

👉 This is your **Stub**

---

# ⚡ What Stub Actually Does (Behind the Scene)

When you call:

```java
accountStub.getAccount(request);
```

👉 Internally:

1. Serialize request → binary
2. Send via HTTP/2
3. Call remote service
4. Wait for response
5. Deserialize response
6. Return Java object

---

# 🔁 Visual Flow

```text
Your Code
   |
   |  accountStub.getAccount()
   v
Stub (Proxy)
   |
   |  HTTP/2 (gRPC)
   v
Remote Service
   |
   v
Response → back to Stub → back to you
```

# ⚡ Types of Stub (VERY IMPORTANT)

## 1. Blocking Stub (You are using this)

```java
AccountServiceBlockingStub
```

✔ Sync call
✔ Waits for response

👉 Easy & common

---

## 2. Future Stub

```java
AccountServiceFutureStub
```

✔ Async (Future-based)

---

## 3. Async Stub

```java
AccountServiceStub
```

✔ Streaming
✔ Non-blocking
✔ Advanced

# 🧠 Why Stub is Powerful

Without stub ❌:

```text
You need:
- HTTP client
- Serialization
- Protocol handling
- Error handling
```

With stub ✅:

```java
accountStub.getAccount(request);
```

👉 Looks like normal method call 😍

---

# ⚠️ Important Notes

* Stub is **client-side only**
* Server uses:

```java
extends AccountServiceImplBase
```
