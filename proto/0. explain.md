```proto
syntax = "proto3";

option java_multiple_files = true;
option java_package = "snvn.grpc";

service AccountService {

  rpc GetAccount (AccountRequest)
      returns (AccountResponse);

}

message AccountRequest {
  int64 userId = 1;
}

message AccountResponse {
  int64 accountId = 1;
  string status = 2;
}
```

তোমার `.proto` ফাইলটা **gRPC service contract**।
এটা দিয়ে **server এবং client দুজনেই একই API structure ব্যবহার করে**। নিচে লাইন-by-লাইন ব্যাখ্যা করছি।

---

# 1️⃣ `syntax = "proto3";`

```proto
syntax = "proto3";
```

এর মানে:

* **Protocol Buffers version 3** ব্যবহার করা হচ্ছে
* gRPC এর modern standard

Proto এর ২টা version আছে:

| Version | Usage                 |
| ------- | --------------------- |
| proto2  | পুরোনো                |
| proto3  | modern (gRPC default) |

---

# 2️⃣ `option java_multiple_files = true;`

```proto
option java_multiple_files = true;
```

এটার কাজ:

Generated Java classes **একটা file এ না হয়ে আলাদা আলাদা file হবে**।

### Without this option

সব এক file এ তৈরি হবে:

```
AccountProto.java
```

### With this option

```text
AccountServiceGrpc.java
AccountRequest.java
AccountResponse.java
```

এটা **clean architecture**।

---

# 3️⃣ `option java_package`

```proto
option java_package = "snvn.grpc";
```

এটার মানে:

Generated Java classes এই package এ যাবে:

```java
package snvn.grpc;
```

Generated classes:

```

snvn.grpc.AccountRequest
snvn.grpc.AccountResponse
```

এটা **Java project structure ঠিক রাখার জন্য দরকার**।

# 5️⃣ RPC method

```proto
rpc GetAccount (AccountRequest) returns (AccountResponse);
```

এটা একটি **remote method**।

মানে:

```
GetAccount(request) → response
```

REST equivalent:

```
GET /api/accounts/{id}
```

gRPC equivalent:

```
GetAccount(AccountRequest) → AccountResponse
```

# 8️⃣ Field numbering

```proto
int64 accountId = 1;
string status = 2;
```

এই number গুলো খুব গুরুত্বপূর্ণ।

কারণ:

```
Protobuf binary serialization এ field number ব্যবহার হয়
```

Example:

| Field     | Number |
| --------- | ------ |
| accountId | 1      |
| status    | 2      |

# 🔟 Server implementation

```java
@GrpcService
public class AccountGrpcService extends AccountServiceGrpc.AccountServiceImplBase {

    @Override
    public void getAccount(AccountRequest request,
                           StreamObserver<AccountResponse> responseObserver) {

        AccountResponse response =
                AccountResponse.newBuilder()
                        .setAccountId(1)
                        .setStatus("ACTIVE")
                        .build();

        responseObserver.onNext(response);
        responseObserver.onCompleted();
    }
}
```
