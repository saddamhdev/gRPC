gRPC-তে **object (message) এর field লেখার নির্দিষ্ট কিছু নিয়ম আছে**। এগুলো `.proto` ফাইলে লেখা হয়। নিচে পরিষ্কারভাবে নিয়মগুলো ব্যাখ্যা করছি।

---

# 1️⃣ Field syntax (মূল structure)

```proto
type fieldName = number;
```

Example:

```proto
message AccountResponse {
  int64 accountId = 1;
  string status = 2;
}
```

এখানে:

| Part        | Meaning      |
| ----------- | ------------ |
| `int64`     | field type   |
| `accountId` | field name   |
| `1`         | field number |

---

# 2️⃣ Field number rule (খুব গুরুত্বপূর্ণ)

```proto
int64 accountId = 1;
```

এখানে `1` হলো **field number**।

### নিয়ম

* প্রতিটি field এর **unique number থাকতে হবে**
* number **change করা যাবে না**
* নতুন field add করলে **নতুন number দিতে হবে**

Example:

```proto
message AccountResponse {
  int64 accountId = 1;
  string status = 2;
  string accountType = 3;
}
```

---

# 3️⃣ Naming convention

gRPC-তে field সাধারণত **camelCase** হয়।

✔ Correct

```proto
int64 accountId = 1;
string accountStatus = 2;
```

❌ Avoid

```proto
int64 ACCOUNT_ID = 1;
string account_status = 2;
```

---

# 4️⃣ Supported data types

Proto3 কিছু built-in types দেয়।

| Type     | Meaning        |
| -------- | -------------- |
| `int32`  | 32 bit integer |
| `int64`  | 64 bit integer |
| `string` | text           |
| `bool`   | true/false     |
| `double` | decimal number |
| `bytes`  | binary data    |

Example:

```proto
message User {
  int64 id = 1;
  string name = 2;
  bool active = 3;
}
```

---

# 5️⃣ Repeated field (List)

List বা array তৈরি করতে `repeated` ব্যবহার করা হয়।

```proto
message UserAccounts {
  repeated AccountResponse accounts = 1;
}
```

Java-তে হবে:

```java
List<AccountResponse>
```

---

# 6️⃣ Nested object

একটা message অন্য message ব্যবহার করতে পারে।

```proto
message UserAccount {
  int64 userId = 1;
  AccountResponse account = 2;
}
```

---

# 7️⃣ Optional field

Proto3-তে সব field optional।
তবে explicit optional লিখতে পারো।

```proto
message AccountResponse {
  int64 accountId = 1;
  optional string status = 2;
}
```

---

# 8️⃣ Default value

Proto3-তে default value automatically set হয়।

| Type   | Default |
| ------ | ------- |
| int    | 0       |
| string | ""      |
| bool   | false   |

Example:

```proto
message AccountResponse {
  int64 accountId = 1;
  string status = 2;
}
```

status না দিলে default হবে:

```
""
```

---

# 9️⃣ Best practice example

```proto
message AccountResponse {

  int64 accountId = 1;

  string status = 2;

  string accountType = 3;

  bool active = 4;

}
```

# 🔟 তোমার project example

```proto
syntax = "proto3";

option java_multiple_files = true;
option java_package = "snvn.grpc";

service AccountService {

  rpc GetAccount (AccountRequest) returns (AccountResponse);

}

message AccountRequest {
  int64 userId = 1;
}

message AccountResponse {
  int64 accountId = 1;
  string status = 2;
}
```
